# deepresearch_summerschool_2026
Датасеты для летней школы.


# Описание файлов

## Задача 1

### Описание данных

Файл _summerschool_task1.csv_ . Разделитель - табуляция.

Каждый элемент колонки **question** - DeepResearch-вопрос, ответ на который искался системой.

Каждый элемент колонки **queries** - запрос к поисковой системе.

Каждый элемент колонки **links** - список ссылок, выданных системой для каждого элемента поля queries.

### Описание заданий

Для разных поисковиков - попробовать для каждого из queries получить список ссылок, оценить процент совпадения и сделать выводы. Задача разминочная, не оценивается.

## Задача 2 

### Описание данных

Файл _summerschool_task2.csv_ . Разделитель - табуляция.

Каждый элемент колонки **url** - это url-адрес сайта, парсинг которого шел.

Каждый элемент колонки **domain** - домен, полученный библиотекой tldextract для данного url-адреса. Отдельно уточняем,что качество работы системы в этой задаче оценивается взвешенно по домену.

Участники для каждого url предоставляют контент, который они спарсили. Контент предобрабатывать не нужно. Финальная оценка будет осуществляться на тестовом файле, который хранится у организаторов. Для обеспечения эффективности оценки настоятельно рекомендуется использование зарубежного / динамического IP.

Выложена только тренировочная часть данных.

## Задача 3 (директория Task3)

### Описание данных

Файл _train_summerschool_task3.csv_ . Разделитель - табуляция.

Каждый элемент колонки **id** - идентификатор ссылки, посещенной моделью.

Каждый элемент колонки **research_question** - исследовательский вопрос, ответ на который искала модель. 

Каждый элемент колонки **link** - ссылка, посещенная моделью.

Каждый элемент колонки **past_query** - запрос, по которому была найдена эта ссылка. (ПРИМЕЧАНИЕ - Вечером 3 июля выложена вторая версия датасета, в которой эта колонка заполнена для ~99.7% примеров)

Каждый элемент колонки **description** - сделанное моделью описание содержимого по ссылке **link** при поиске ответа на вопрос **research_question**. (ПРИМЕЧАНИЕ - В связи с защитой Claude от дистилляции колонка заполнена описаниями от другой, более слабой модели. Значения могут поменяться в будущих версиях датасета.)

Выложена только тренировочная часть данных.

## Задача 4 (директория Task4)

### Описание данных

Файл _train_summerschool_task4.csv_ . Разделитель - табуляция.

Каждый элемент колонки **task3_ids** - список идентификаторов ссылок. Соответствие каждого идентификаторов своей ссылке определяется файлом _summerschool_task3.csv_.

Каждый элемент колонки **research_question** - исследовательский вопрос, ответ на который искала модель. 

Каждый элемент колонки **query** - запрос, сгенерированный поисковой системой на основании содержания ссылок из **task3_ids**.

Каждый элемент колонки **query_ids** - то, каким по счету модель делала запрос query. Полезно для восстановления хронологии диалога.

Выложена только тренировочная часть данных.
